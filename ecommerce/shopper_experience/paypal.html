Paypal доступен и популярен по всему миру. Он не взимает плату за подписку, а создать аккаунт очень легко. Вот почему мы рекомендуем его для начинающих в Odoo. Он работает бесперебойно, когда клиент перенаправляется на сайт Paypal для регистрации платежа.

Аккаунт Paypal

Бизнес-счет нужен для получения платежей с Paypal. Создайте [UNKNOWN NODE problematic]Бизнес-счет Paypayl <https://www.paypal.com/us/merchantsignup/applicationChecklist?signupType=CREATE_NEW_ACCOUNT&productIntentId=wp_standard> [UNKNOWN NODE problematic]__ или обновите ваш Персональный счет на Бизнес. Перейдите в настройки Paypal и нажмите * Обновить на Бизнес-счет [UNKNOWN NODE problematic]. Затем продолжите несколько шагов настройки.

Настройки в Paypal

Сначала, давайте посмотрим, как настроить ваш аккаунт Paypal, чтобы создать бесперебойный опыт работы клиента с Odoo.
Войдите и откройте настройки. Перейдите в * Товары и Услуги&gt; Платежи сайта * и нажмите * Обновить * в * Преимущества сайта [UNKNOWN NODE problematic].

автоматический возврат

Auto Return automatically redirects your customers to Odoo once the payment is processed. Check Auto Return and enter your domain name «/shop/confirmation» as Return URL (e.g. https://yourcompany.odoo.com/shop/confirmation).

Эта URL-адрес запрашивает Paypal, но не используется на практике, поскольку Odoo передает ее на каждой транзакции. Не волнуйтесь, если вы управляете несколькими каналами продаж или базами данных Odoo.

Перевод данных об оплате (PDT)

Payment Data Transfer delivers the payment confirmation to Odoo as soon as it is processed. Without it, Odoo cannot end the sales flow. This setting must be activated as well. When saving, an Identity Token is generated. You will be later requested to enter it in Odoo.

Опциональный счет Paypal

Мы советуем не побуждать клиентов входить с помощью аккаунта Paypal, когда они получают оплату. Пусть они также платят дебетовыми / кредитными карточками, иначе вы можете потерять некоторые сделки. Убедитесь, что этот параметр.

Мгновенные оповещения оплаты (IPN)

PDT направляет подтверждение заказов только один раз. В результате ваш сайт должен запускаться, когда это случается; в противном случае он никогда не получит сообщение. Поэтому мы советуем активировать * Мгновенные уведомления оплаты * (IPN) в горе. С IPN, доставка подтверждение заказов практически гарантировано с тех пор, как IPN повторно направляет подтверждения пока ваш сайт не подтвердит получение.

Чтобы активировать IPN, вернитесь в меню * Платежи сайта * и нажмите * Обновить * в * Моментальное уведомление оплаты [UNKNOWN NODE problematic].
The Notification URL to set is your domain name + “payment/paypal/ipn” (e.g. https://yourcompany.odoo.com/payment/paypal/ipn).

Формат сообщений оплаты

If you use accented characters (or anything else than basic Latin characters) for your customer names, addresses… you MUST configure the encoding format of the payment request sent by Odoo to Paypal.

Опасно

If you don’t configure this setting, some transactions fail without notice.

To do so, open:

  • this page for a test account
  • this page for a production account

Then, click More Options and set the two default encoding formats as UTF-8.

Ваш аккаунт Paypal готов!

Совет

If you want your customers to pay without creating a Paypal account, Paypal Account Optional needs to be turned on.

Настройки в Odoo

Активация

Активируйте * Paypal * с панели настроек Продаж, Выставление счетов и Электронной коммерции, или из меню настройки * платежных эквайеров [UNKNOWN NODE problematic].

Полномочия

Odoo требует трех учетных данных Paypal:

  • Email ID is your login email address in Paypal.
  • Merchant ID can be found in the settings of your Paypal account, in Profile > About the business.
  • Paypal PDT Token is given in Website payments configuration as explained here above.

оплата транзакции

Вы можете взимать дополнительную комиссию с клиентов за оплату через Paypal; Это покрывает взыскании с вас комиссии за транзакцию Paypal. После направления на Paypal, ваш клиент видит дополнительную сумму к сумме заказа.

Чтобы активировать это, перейдите на вкладку Настройка Paypal а Odoo и отметьте * Добавить дополнительные комиссии [UNKNOWN NODE problematic].

Вы можете перейти на [UNKNOWN NODE problematic]Комиссию Paypal <https://www.paypal.com/webapps/mpp/paypal-fees> [UNKNOWN NODE problematic]__ чтобы установить комиссию.

..note :: [UNKNOWN NODE problematic]Продавцам в ЕС <https://europa.eu/youreurope/citizens/consumers/shopping/pricing-payments/index_en.htm> [UNKNOWN NODE problematic]__ запрещено взимать дополнительную комиссию за оплату кредитными карточками.

Перейдите в онлайн!

Ваши настройки готовы к запуску. Убедитесь, что включен режим * Разработчик [UNKNOWN NODE problematic]. Затем опубликуйте метод платежа, нажав на кнопку * Опубликовано * рядом.

Теперь Paypal доступен в ваших платежах в модулях Электронной коммерции, продажи и выставления счетов. Клиенты перенаправляются на сайт Paypal при нажатии * Оплатить сейчас [UNKNOWN NODE problematic]. Они вернутся на страницу подтверждения в Odoo после того, как платеж будет обработан.

тестовое среду

Вы можете протестировать процесс оплаты в Odoo благодаря счетам Paypal Sandbox.

Войдите на [UNKNOWN NODE problematic]Сайт разработчика Paypal <https://developer.paypal.com/> [UNKNOWN NODE problematic]__ с вашими учетными данными Paypal.

Это создаст два счета sandbox:

  • Бизнес-счет (для использования в качестве покупателей, например. [UNKNOWN NODE problematic]pp.merch01-facilitator@example.com <mailto:pp.merch01-facilitator@example.com> [UNKNOWN NODE problematic]__).
  • Персональный счет по умолчанию (для использования в качестве покупателя, eg [UNKNOWN NODE problematic]pp.merch01-buyer@example.com <mailto:pp.merch01-buyer@example.com> [UNKNOWN NODE problematic]__).

Войдите в Paypal Sandbox учетной записью продавца и следуйте тем самым инструкциям настройки. Введите учетные данные sandbox в Odoo и убедитесь, что Paypal сих пор не установлено как * Тестовое среду [UNKNOWN NODE problematic]. Также убедитесь, что автоматическое выставление счета не активирован в ваших настройках Электронной коммерции, чтобы не создавать счета когда завершаются фиктивные транзакции.

Запустите тестовую транзакцию в Odoo, используя личный аккаунт Sandbox.
