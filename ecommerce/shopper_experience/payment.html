Odoo включает несколько способов оплаты с электронной коммерции, продаж и счетов-фактур.

Какие доступные способы оплаты?

банковский перевод

Банковский перевод - это доступный способ оплаты по умолчанию. Цель - предоставить своим клиентам свои банковские реквизиты, чтобы они могли платить самостоятельно через свой банк. Это очень легко начать, но медленно и неэффективно. Выберите онлайн-покупателей, как только сможете!

Платежные эквайеры

Переадресовывайте своих клиентов на платформы, чтобы легко получать деньги и отслеживать статус платежа (обратная связь). Odoo поддерживает все больше и больше платформ:

  • Paypal
  • Ingenico
  • Authorize.net
  • Adyen
  • Buckaroo
  • PayUmoney
  • Sips
  • Stripe

Как это воплотить

После того, как способ оплаты готов, сделайте его видимым в интерфейсе платежа и использование режима ** Производство [UNKNOWN NODE problematic].

Как разрешить клиентам сохранять и повторно использовать кредитные карточки

Чтобы облегчить уплату возвращенных клиентов, вы можете позволить им сохранять и повторно использовать кредитную карту, если они хотят. Если так, токен платежа будет сохранен в Odoo. Этот параметр доступен для Ingenico и Authorize.net.

Вы можете включить это с помощью формы настройки эквайера.

Как дебетировать кредитные карты для оплаты подписки

[UNKNOWN NODE problematic]Подписка Odoo <https://www.odoo.com/page/subscriptions> [UNKNOWN NODE problematic]__ позволяет автоматически оплачивать услуги на регулярной основе. Наряду с этим, вы можете иметь автоматическое дебетования кредитной карты клиента.

Этот параметр доступен для Ingenico и Authorize.net.

Вы можете включить это с помощью формы настройки эквайера.

Таким образом, платежный токен будет фиксироваться, когда пользователь перейдет на подписку, а автоматический дебет запускаться каждый раз, когда счет-фактура выпадает из подписки.

Как пользоваться другими эквайерами (расширенные)

Odoo может предоставить запросы на единовременную оплату и переадресацию к любому плательщика. Но нет обратной связи, то есть Odoo не отслеживаем состояние транзакции. Таким образом, вы подтвердите заказ вручную, как только вы получите плату.

Как это сделать:

  • Переключиться в режим разработчика.
  • Установите метод платежа ** кастомных [UNKNOWN NODE problematic].
  • Настройте форму для оплаты (шаблон формы S2S), как указано вашим платежным эквайером. Вы можете начать с * default_acquirer_button [UNKNOWN NODE problematic], которое вы можете дублировать.

другие настройки

Система Odoo также может быть использована для более продвинутых платежных транзакций, таких как платы за рассрочку (напр., [UNKNOWN NODE problematic]Paypal Installment Plans <https://developer.paypal.com/docs/classic/paypal-payments-standard/integration-guide/installment_buttons> [UNKNOWN NODE problematic]__).

Такая служба настройки осуществляется по требованию наших технических экспертов на основе ваших собственных требований. Бизнес-консультант может связаться с вами с таким вопросом. [UNKNOWN NODE problematic]Свяжитесь с нами. <https://www.odoo.com/page/contactus> [UNKNOWN NODE problematic]__
