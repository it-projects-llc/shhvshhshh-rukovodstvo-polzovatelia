Варианты товара используются для управления товарами, имеющими разные варианты, например размер, цвет и т. Это позволяет управлять товаром на уровне шаблона (для всех вариантов) и на уровне вариантов (специфические атрибуты).

Например, компания, продающая футболки, может иметь такой товар:

  • Футболка B &amp; C

    • Размеры: S, M, L, XL, XXL
    • Цвета: синий, красный, белый, черный

В этом примере ** Топ B &amp; C ** называется шаблоном товара, а ** Топ B &amp; C, S [UNKNOWN NODE problematic], синий - это вариант. Размеры и цвет является ** атрибутами [UNKNOWN NODE problematic].

В приведенном выше примере в общей сложности 20 различных товаров (5 размеров на 4 цвета). Каждый из этих товаров имеет собственную инвентаризацию, продажа и др.

влияние вариантов

  • ** Штрих-код ** код и штрих-код ассоциируются с вариантом, а не с шаблоном. Каждый вариант может иметь свой штрих-код / ​​SKU.
  • ** Цена [UNKNOWN NODE problematic]: каждый вариант товара имеет свою публичную цену, которая исчисляется на основе цены шаблона ($ 20) за дополнительным аксессуаром для каждого варианта (+ 3 долл. США за красный цвет). Тем не менее, вы можете определить ценовые правила, применяемые к шаблону или варианта.
  • ** фото [UNKNOWN NODE problematic]: запас руководствуется вариантом товара. Вы не владеете футболками, у вас есть только &quot;Футболки, S, красный&quot; или &quot;Футболки, M, синий&quot;. Для информационного назначения, в форме шаблона товара вы получаете запас, который является суммой каждого варианта (но фактический запас исчисляется по варианту).
  • ** Изображения [UNKNOWN NODE problematic]: картинка связана с вариантом, каждая вариация товара может иметь свою основную картину.
  • ** Другие поля [UNKNOWN NODE problematic]: большинство других полей принадлежит к шаблону товара. Если вы их обновляете, он автоматически обновляет все варианты (пример: Счет дохода, Налоги).

Стоит ли использовать варианты?

Когда следует использовать варианты?

Использование вариантов имеет следующие последствия:

  • ** Электронная коммерция [UNKNOWN NODE problematic]: в вашем интернет-магазине клиент увидит только шаблоны товаров на странице каталога. Когда посетитель нажимает на такой товар, он будет иметь варианты выбора среди вариантов (цвета, размеры, …)
  • ** Производство ** Использование вариантов позволяет определить лишь одну спецификацию для шаблона товара и небольшие вариации для некоторых вариантов. Пример: вместо создания спецификации для &quot;Футболки, красный, S&quot; вы создаете спецификацию для &quot;футболки&quot; и добавляете строки, соответствующие конкретному размера S, а также другие строки, специфичные для красного цвета.
  • ** Ценообразование [UNKNOWN NODE problematic]: цена товара по умолчанию вычисляется по цене шаблона товара и добавляет необязательную дополнительную цену за каждым размером варианта. Таким образом, варианты цены проще поддерживать, поскольку вам не нужно устанавливать цены по каждому варианту. Однако можно создать ценовые правила, чтобы установить цену за вариантом.

Когда следует избегать использования вариантов?

Использование вариантов может добавить уровень сложности в том, как вы используете Odoo. Вы должны рассмотреть возможность использования вариантов только в том случае, когда вам это нужно, чтобы уменьшить сложность управления многими товарами, которые являются аналогами.

Например, импорт начального каталога товаров является более сложным, если вы используете варианты. Вы не можете просто импортировать список товаров, вы должны импортировать шаблоны товаров и все связанные с ними варианты.

Кроме того, вы также должны тщательно выбрать параметры, которыми вы управляете как отдельными шаблонами товаров, и те, что являются вариантами. Например, компания, которая имеет эти товары:

  • Качество: футболки, поло
  • Цвет: красный, синий
  • Размер: S, M, L, XL

В таком случае вы можете создать 1 шаблон с тремя параметрами вариантов (Компоновка, футболки, поло). Но рекомендуется создать два разных шаблоны товаров, поскольку футболки могут сильно отличаться от поло или рубашек, и потребитель надеется увидеть их как два разных товара в электронной коммерции:

  • Шаблон товара: футболка

    • Цвет: красный, синий
    • Размер: S, M, L, XL

  • Шаблон товара: поло

    • Цвет: красный, синий
    • Размер: S, M, L, XL

Настройка

Активизируйте функцию варианта

Прежде чем использовать варианты товара, сначала нужно активировать варианты товара в настройках. Для этого нужно перейти к приложению Продажи. В меню: menuselection: [UNKNOWN NODE title_reference] найдите строку ** Варианты товара ** и поставьте флажок ** Товары могут иметь несколько атрибутов [UNKNOWN NODE problematic], затем нажмите ** Применить [UNKNOWN NODE problematic].

Создание продуктов с различными вариантами

После активации вариантов, вы можете добавить варианты в ваших товаров. Для этого перейдите к модулю: menuselection: [UNKNOWN NODE title_reference]. Он также доступен в модуле Покупки и Состав.

Теперь нажмите на товар, к которому нужно добавить варианты.

На странице товара появилась новая вкладка Варианты. Фиолетовый номер, написанный сверху, - это количество вариантов этого товара в настоящее время. Чтобы добавить новые варианты, нажмите на плитку. В новом окне нажмите кнопку ** Создать [UNKNOWN NODE problematic].

В разделе ** Атрибуты ** щелкните на развернутом меню и выберите тип разницы, какой вы хотите добавить. Если вариант еще не существует, вы можете создать его на лету, нажав кнопку Создать и изменение …

В окне Атрибуты поле ** Значение ** - это описание атрибута, такого как Зеленый, Пластиковый или «32 Гб». Поле ** Атрибут ** - это тип варианта, такого как Цвет, Материал или Память.

Вы можете добавить стоимость варианта на лету, добавив его в поле ** Дополнительные характеристики ** или выберите позже. Нажмите кнопку ** Сохранить [UNKNOWN NODE problematic].

Вы также можете добавить другой штрих-код и внутренняя ссылка на вариант.

Когда вы ввели все спецификации этого варианта, нажмите кнопку ** Сохранить [UNKNOWN NODE problematic].

Введение

Приведенные ниже примеры являются основой этого шаблона товара, который имеет два атрибуты варианта:

  • Футболка B &amp; C

    • Цвет: красный, синий, белый
    • Размер: S, M, L, XL, XXL

Управление возможностями комбинации

По умолчанию, используя указанный выше шаблон товара, вы получаете 15 различных товаров (3 цвета, 5 размеров). Если размер XXL существует только для красных и синих футболок, вы можете отключить вариант белого товара.

Для этого нажмите кнопку ** Варианты [UNKNOWN NODE problematic], выберите XXL, белая футболка. Из формы товара снимите обозначения ** Активно ** футболки белого цвета XXL.

Совет

Отключение товара отличается от наличия инвентаризации 0.

Установление цены за вариант

Вы можете добавить стоимость по основной цене для некоторых вариантов товара.

После активации вариантов, вы можете добавить варианты в ваших товаров. Для этого перейдите к модулю Продажи, откройте: menuselection: [UNKNOWN NODE title_reference] и нажмите на товар, который нужно изменить. Нажмите кнопку ** Варианты цен [UNKNOWN NODE problematic], чтобы посмотреть список вариантов значений.

Нажмите на названии варианта, к которому нужно добавить значение, чтобы сделать 3 поля редактируемыми. В поле ** Атрибут дополнительной характеристики ** добавьте стоимость варианта, который будет добавлен к начальной цены.

Когда вы введете все дополнительные значения, нажмите кнопку ** Сохранить [UNKNOWN NODE problematic].
