Введение

Электронное выставления счетов для Колумбии можно получить из Odoo 12 и требует следующих модулей:

  1. ** l11n_co ** Все основные данные для управления модулем бухгалтерского учета, содержащие стандартные настройки для: плана счетов счетов, налогов, ретенций, типов идентификационных документов
  2. ** l10n_co_edi ** Этот модуль включает в себя все дополнительные поля, необходимые для интеграции с T &amp; S Carvajal, и генерирования электронного счета-фактуры на основе правовых требований DIAN.

Рабочий процесс

Настройка

1. Установите модули колумбийской локализации

Для этого перейдите в Приложения и найдите Колумбию. Затем нажмите кнопку Установить для первых двух модулей.

2. Настройте учетные данные для веб-сервиса Carvajal T &amp; S

После того, как модули установлены, для того, чтобы иметь возможность соединяться с веб-службой Carvajal T &amp; S, необходимо настроить пользователя и учетные данные, эта информация будет предоставлена Carvajal T &amp; S.

Перейдите к: menuselection: [UNKNOWN NODE title_reference] и найдите раздел * Колумбийские электронные счета [UNKNOWN NODE problematic].

Используя режим тестирования, можно подключиться к среде тестирования Carvisal T &amp; S. Это позволяет пользователям проверять полный рабочий процесс и интеграцию с порталом CEN Financiero, доступным здесь: https://cenfinancierolab.cen.biz

После того, как Odoo и Carvajal T &amp; S полностью настроены и готовы к производству, среда тестирования может быть отключен.

3. Настройте ваши данные отчета

Как часть настроенной информации, направляется в XML, можно определить данные для фискального раздела и банковскую информацию в PDF-файле.

Перейдите к: menuselection: [UNKNOWN NODE title_reference] и найдите раздел * Колумбийские электронные счета [UNKNOWN NODE problematic].

4. Установите данные, необходимые в XML

4.1 Партнер

4.1.1 Идентификация

Как часть колумбийской локализации, типа документов, определенные DIAN, теперь доступны в форме Партнера. Колумбийские партнеры должны иметь свой идентификационный номер и тип документа:

Совет

Когда тип документа RUT, идентификационный номер должен быть настроен в Odoo, включая проверочную цифру, Odoo будет разделять этот номер, когда передаются данные третьей стороне.

4.1.2 Фискальная структура (RUT)

Коды ответственности партнера (раздел 53 документа RUT) включены как часть модуля электронного счета, который является частью информации, требуется DIAN.

Эти поля можно найти в: menuselection: [UNKNOWN NODE title_reference]

Дополнительно было добавлено два булевых поля для определения фискального режима партнера.

4.2 Налоги

Если ваши операции по продаже включают товары с налогами, важно учитывать, что дополнительное поле * Тип значения * необходимо настроить на налог. Этот параметр находится на вкладке Дополнительные параметры.

Типы налога на содержание (ICA, IVA, Fuente) также включены в опций настройки налогов. Эта конфигурация используется для корректного отображения налогов в формате PDF.

4.3 Журналы

После того, как DIAN назначил официальную последовательность и префикс для решения электронных счетов, журналы продаж, связанные с документами на счете-фактуре, нужно обновлять в Odoo. Последовательности можно обращаться, используя режим разработчика:: menuselection: [UNKNOWN NODE title_reference].

После того, как последовательность открыта, поля Приставка и Следующий номер должны быть настроены и синхронизированы с CEN Financiero.

1. Счет

1.1 Создание счета

Функциональный рабочий процесс, который происходит перед проверкой счета-фактуры, не меняется. Основными изменениями, которые вводятся в электронный счет, являются следующие поля:

Есть три типа документов:

  • ** Factura Electronica ** Этот документ является обычным типом и применяется к счетам-фактурам, сторно и дебетовое возвращения.
  • ** Factura de Importación [UNKNOWN NODE problematic]: этот параметр для операций импорта.
  • ** Factura de contingencia [UNKNOWN NODE problematic]: Это исключительный тип, который используется как ручное резервное копирование в случае, если компания не может использовать ERP, и необходимо генерировать счет вручную, когда этот счет добавлен к ERP, этот тип счета-фактуры следует выбрать.

1.3 Получение правовых XML и PDF документов

Поставщик электронного счета получает XML-файл и продолжает проверку структуры и информации в нем, если все правильно, изменение статуса счета-фактуры на &quot;Проверено&quot; после использования кнопки &quot;Проверить состояние Carvajal&quot; в выпадающем меню Действие. Затем они начинают генерировать правовой XML, включая цифровую подпись и уникальный код (CUFE), а также генерируется фактура PDF, которая включает QR-код и CUFE.
