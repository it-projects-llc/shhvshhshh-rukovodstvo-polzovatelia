SEPA, единственная область евро-платежей, является платежно-интеграционной инициативе Европейского Союза по упрощению банковских переводов, выраженных в евро. С помощью ** Прямого дебета SEPA ** ваши клиенты могут подписать ** мандат [UNKNOWN NODE problematic], который дает вам возможность автоматически собирать будущие платежи со своих банковских счетов.

Вы можете записать ваши клиентские мандаты в Odoo и создать XML-файлы, содержащие платежи в ожидании, сделанные с мандатом SDD.

Примечание

SDD поддерживается всеми странами SEPA, включающий 28 стран-членов Европейского Союза, а также дополнительные страны.

Настройка

Перейдите в: menuselection: [UNKNOWN NODE title_reference] и активируйте функцию ** Прямого дебета SEPA (SDD) [UNKNOWN NODE problematic]. Введите ** Идентификатор кредитора ** вашей компании. Этот номер предоставляется вашим банком или органом, ответственным за их поставки.

Мандаты прямого дебета SEPA

создайте

Мандат прямого дебета SEPA - это документ, который подписывают ваши клиенты для авторизации вас для получения денег прямо с их банковских счетов.

Чтобы создать новый, перейдите в: menuselection: [UNKNOWN NODE title_reference], нажмите на * Создать * и заполните новую форму мандата.

Сначала, экспортировать PDF-файл, нажав * Печать . Затем ваш клиент должен подписать этот документ. После завершения загрузите подписанный файл в поле * Оригинальный документ ** и нажмите * Проверить [UNKNOWN NODE problematic], чтобы приступить к выполнению мандата.

Совет

Сначала убедитесь, что ** Детали банковских счетов IBAN ** верно записанные на форме контакта дебитора, на вкладке * Бухучет * и в ваших собственных настройках * Банковского счета [UNKNOWN NODE problematic].

Закройте или отмените мандат

Мандаты прямого дебета закрываются автоматически после их * Дать окончания [UNKNOWN NODE problematic]. Если это поле оставить пустым, мандат продолжает быть * Активным [UNKNOWN NODE problematic], пока не будет * Закрыто * или * Отменены [UNKNOWN NODE problematic].

Если нажать ** Закрыть [UNKNOWN NODE problematic], обновляется конечный день мандата к текущему дню. Это означает, что счета-фактуры, сделанные после этого дня, не разделаются автоматически с помощью SDD-платежа.

Нажав на ** Отозвать [UNKNOWN NODE problematic], немедленно отключается мандат. Платеж SDD больше не может быть зарегистрирован, независимо от даты счета-фактуры. Однако платежи, которые уже были зарегистрированы, все равно будут включены в следующий XML-файла SDD.

Примечание

После того, как мандат будет * закрыто * или * отозвано [UNKNOWN NODE problematic], его можно будет реактивировать. Если клиент снова потребует оплаты с помощью SDD, вам придется создать новый мандат с нуля.

Получайте платежи с Групповыми платежами SDD

Счета заказчику

Все новые опубликованы счета, выданные клиентам с активным мандатом, будут автоматически регистрировать платеж и обозначать их статус как * Ополчения [UNKNOWN NODE problematic].

Совет

Если у вас есть неоплаченные счета, которые могли быть оплачены с новым мандатом, это все еще возможно сделать. Перейдите в счет, нажмите * Зарегистрировать платеж * и выберите * Прямой дебет SEPA * как метод платежа.

Создайте XML-файлы Прямого дебета SEPA, чтобы предоставить платежи

Чтобы предоставить ваши платежи SDD вашем банке, вам сначала нужно создать ** XML-файл [UNKNOWN NODE problematic], который можно загрузить прямо в вашем банковском интерфейсе.

Файлы, созданные Odoo, следуют по спецификациям Прямого дебета SEPA ** PAIN.008.001.02 [UNKNOWN NODE problematic], которые требуются Руководством по внедрению клиента в банк SEPA, что обеспечивает совместимость с банками.

Чтобы создать XML-файл для ожидаемых платежей SDD, перейдите на связанный * банковский журнал * на вашей * панели приборов Бухучета * нажмите * Платежи прямого дебета для получения [UNKNOWN NODE problematic].

Выберите все платежи в списке, которые вы хотите включить в ваш XML-файл SDD нажмите * Действие * и выберите * Создать групповой платеж [UNKNOWN NODE problematic].

Затем Odoo направит вас на форму вашего * Группового платежа [UNKNOWN NODE problematic]. Нажмите на * Подтвердить * и загрузите XML-файл SDD.

И наконец загрузите этот файл в ваш банк для обработки платежей.

Совет

Вы можете получить все созданные XML-файлы SDD, перейдя в: menuselection: [UNKNOWN NODE title_reference].
